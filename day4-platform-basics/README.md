# Day 4 – Salesforce Flow & Automation Basics

## What is Flow Builder?
Flow Builder is a Salesforce tool used to automate tasks without writing code. It works like a visual flowchart where users can create automation using drag-and-drop elements.

## Why Automation Matters
Automation helps businesses save time, reduce manual work, improve accuracy, and increase productivity. Instead of doing repetitive tasks manually, Salesforce can perform them automatically.

## Different Automation Types
Salesforce provides different automation tools such as:
- Flow Builder
- Process Automation
- Approval Processes
- Record Triggered Flows

These tools help automate business processes efficiently.

## Basic Flow Types

### 1. Screen Flow
A flow that shows screens to users and collects input data.

### 2. Record-Triggered Flow
Runs automatically when a record is created, updated, or deleted.

### 3. Autolaunched Flow
Runs automatically in the background without user interaction.

## What is Record-Triggered Flow?
A Record-Triggered Flow automatically executes when changes happen to Salesforce records. It is commonly used for updating fields, sending emails, creating tasks, and automating business processes.

## When Flows Execute
Flows can execute:
- Before saving a record
- After saving a record
- On record creation
- On record update
- On record deletion

## Real-World Examples
- Automatically send a welcome email when a new customer is added.
- Create a task when a support case is opened.
- Update account status when payment is completed.

## Visual Overview of Flow Builder
Flow Builder contains:
- Elements → Actions performed in the flow
- Connectors → Connect different steps
- Canvas → Workspace where flows are designed

## Beginner Understanding
Flow Builder helps users automate processes easily using a visual interface without coding knowledge. It improves business efficiency and simplifies repetitive tasks.
